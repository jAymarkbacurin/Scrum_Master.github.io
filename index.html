Project Overview

This progress report covers the second sprint of the School Portal Project. The team has focused on developing the backend and setting up the database structure. The project aims to create a centralized system for managing academic records, announcements, and student-teacher communication.

Sprint Progress

  • Sprint 1: Gathered requirements and conducted initial planning.
  • Sprint 2: Completed backend setup and basic database structure.
  • Sprint 3: Developed the user authentication module and tested for security.

Sprint Summary

Sprint Goals:

  • Develop a responsive design for the website.
  • Complete the database structure for the school portal.
  • Application Programming Interface integration

Challenges

  • During the second sprint, the team faced difficulties in integrating third-party APIs. These issues were resolved through additional research and team collaboration.
  • The team also faced challenges in designing a responsive and user-friendly interface for the school portal. This was addressed through regular design reviews and user feedback sessions.
  • One of the challenges faced by the team was the limitation of computer resources needed to complete the project.
